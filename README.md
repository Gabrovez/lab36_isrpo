## Описание проекта

TaskBoard — это полнофункциональное веб-приложение для управления задачами, реализованное с использованием ASP.NET Core Web API на бэкенде и чистого JavaScript на фронтенде.

### Основные возможности:

- ✅ **Просмотр задач** — отображение списка всех задач с информацией о названии, описании, дате создания и статусе выполнения
- ➕ **Добавление задач** — создание новых задач с указанием названия и описания
- ✏️ **Редактирование** — изменение названия и описания существующих задач прямо в интерфейсе
- ✔️ **Отметка выполнения** — возможность отметить задачу как выполненную (визуально перечёркивается)
- 🗑️ **Удаление задач** — удаление ненужных записей с подтверждением
- 🔍 **Фильтрация** — просмотр всех задач, только активных или только выполненных
- 📊 **Счётчик** — отображение количества выполненных задач от общего числа

## Инструкция по запуску

### Запуск бэкенда

1. Откройте терминал и перейдите в папку проекта:
```bash
cd TaskBoardApi
```

2. Восстановите зависимости (если необходимо):
```bash
dotnet restore
```

3. Запустите сервер:
```bash
dotnet run
```

Сервер запустится на порту 5000. Swagger документация доступна по адресу:  
`http://localhost:5000/swagger`. В строке  будет написан адрес на котором он запустился не забудьте поменять.

### Запуск фронтенда

- Установите расширение "Live Server"
- Нажмите правой кнопкой мыши на `frontend/index.html`
- Выберите "Open with Live Server"

## Главные выводы

1. **REST API и CRUD-операции** — научился создавать полноценный Web API с использованием всех HTTP-методов (GET, POST, PUT, DELETE). Каждый метод решает свою задачу: GET получает данные, POST создаёт, PUT обновляет, DELETE удаляет.

2. **Entity Framework Core** — освоил работу с ORM: создание моделей с атрибутами валидации (`[Required]`, `[MaxLength]`), контекста базы данных (`DbContext`) и миграций для автоматического создания таблиц в SQLite.

3. **Асинхронное программирование** — понял важность `async/await` при работе с базой данных и сетевыми запросами. Методы `ToListAsync()`, `FindAsync()`, `SaveChangesAsync()` не блокируют поток выполнения.

4. **Fetch API и взаимодействие с сервером** — научился отправлять асинхронные запросы с фронтенда, обрабатывать JSON-ответы и корректно обрабатывать ошибки через `!response.ok`. Модуль `api.js` отделяет сетевую логику от интерфейса.

5. **Dependency Injection и CORS** — разобрался с внедрением зависимостей через конструктор контроллера (`AppDbContext db`) и настройкой кросс-доменных запросов для связи фронтенда и бэкенда.
